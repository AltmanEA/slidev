## План разработки

### Этап 1: Настройка проекта расширения (30 мин)
1. Создание структуры VS Code extension проекта
2. Настройка package.json с зависимостями (slidev, vsce)
3. Конфигурация TypeScript и build процесса
4. Базовая активация расширения

### Этап 2: Core логика - файловая система (45 мин)
1. **Lecture Discovery Service** ✅
   - Сканирование директории slides/
   - Обнаружение лекций по slides.md
   - Валидация структуры лекции

2. **Frontmatter Parser** ✅
   - Парсинг YAML frontmatter из slides.md
   - Извлечение title, description, date
   - Валидация обязательных полей

3. **Course Index Generator** ✅
   - Генерация slides.json из обнаруженных лекций
   - Сортировка и структурирование данных
   - Детерминированный порядок лекций

### Этап 3: Slidev интеграция (30 мин)
1. **Lecture Environment Manager**
   - Инициализация Node.js проекта в директории лекции
   - Локальная установка slidev
   - Изоляция зависимостей между лекциями

2. **Build System**
   - Команды разработки (dev server)
   - Команды сборки (статический build)
   - Обработка ошибок сборки

### Этап 4: UI реализация (60 мин)
1. **Tree Data Provider**
   - Course Node (корень курса)
   - Lecture Group Node (группировка)
   - Lecture Node (отдельная лекция)

2. **View Container и Commands**
   - Регистрация в Activity Bar
   - Команды: открыть лекцию, dev server, build
   - Refresh логика

### Этап 5: Интеграция и тестирование (30 мин)
1. Связывание UI с core логикой
2. Обработка edge cases (ошибки, отсутствующие файлы)
3. Тестирование с образцом курса

### Этап 6: Полировка и документация (15 мин)
1. Error handling и user feedback
2. README с инструкциями
3. Иконки и метаданные
