# Работа ИИ агента

Пиши код как опытный разработчик:
- Используй ясные, самодокументирующиеся имена.
- Избегай магии — сложных паттернов и синтаксиса, которые могут затруднить понимание кода.
- Комментируй сложные участки, особенно если используется нестандартное решение или сложная логика.
- Если код слишком сложный или объемный (больше 50 строк), разбей его на подзадачи или предложи упрощённое решение, которое будет легче понять.
- Не используй шаблоны проектирования, если они не добавляют явных преимуществ (например, если задача не требует расширяемости или модульности).

# Среда разработки

Агент работает в vscode, плагин koda. 
Система windows, использовать в терминале команды powershell.
Терминал запускается в папке проекта, команда cd не нужна.
Несколько команд разделяются ";"
Аргументы команд (например, пути) нужно заключать в кавычки.

# Структура проекта

Документация находится в папке docs/, исходные коды в src/, тесты в src/test/.
Используется pnpm.

# Решение задачи.

1. Ознакомься с документацией и уточни требования.
2. Согласуй с пользователем план решения, определив ключевые этапы.
3. При необходимости разбей задачу на подзадачи и уточни приоритеты.
4. После выполнения задачи — добавь тесты и проверь функциональность.
5. После успешного тестирования — обнови документацию.

# Решение проблем и исправление ошибок

1. Проанализируй проблему и составь список наиболее вероятных источников ошибок.
2. Оцени варианты решения с точки зрения:
   - Минимизации изменений в коде
   - Увеличения стабильности и тестируемости
   - Влияния на производительность
3. Реализуй оптимальный вариант решения.
4. Проверь исправление с помощью тестов.
5. Если требуется, обнови документацию.
